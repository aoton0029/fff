## 1. プロジェクト概要

本システムは、社内で使用されているExcelファイルに記録された給与・配賦・労務費振替・応援連絡票データを、Webブラウザ経由でアップロードし、データベースへ一元管理するとともに、SAP連携用の出力ファイルを自動生成するWebアプリケーションである。

### 1.1 システム名称

| 項目 | 内容 |
|------|------|
| 正式名称 | エクセルデータ取り込みシステム |
| 略称 | Excel取り込みシステム |
| 開発言語 | Python 3.14+ |
| フレームワーク | Flask 3.1+ |
| 対象環境 | Windows (開発：SQLite、本番：SQL Server) |

## 2. 背景・目的

### 2.1 背景

これまで、給与データ・工程配賦データ・労務費振替依頼書・応援連絡票は、各担当者がExcelファイルで管理し、SAP（基幹システム）への入力は手作業で行っていた。このため、以下の課題が存在していた。

- **入力工数の増大:** ExcelからSAPへの転記に多大な時間を要する
- **転記ミスのリスク:** 手作業による誤入力が発生しやすい
- **データの散在:** 各担当者のPC上にファイルが分散し、進捗管理が困難
- **一元管理の欠如:** 月次処理状況の可視化ができていない

### 2.2 目的

| No | 目的 |
|----|------|
| 1 | Excelファイルのアップロードによるデータ取り込み自動化 |
| 2 | Pydanticを用いた入力データのバリデーションによる品質確保 |
| 3 | データベースへの一元管理と月次処理状況の可視化 |
| 4 | SAP連携用TSVファイルの自動生成による転記作業の廃止 |
| 5 | マスタデータ（課コード・変換マスタ）のWeb管理 |

## 5. 業務フロー

### 5.1 月次処理フロー全体

```
月次処理開始
    │
    ├─ [1] 処理年月の設定（全データクリア）
    │
    ├─ [2] 給与データ取り込み
    │       └── 給与Excel → アップロード → バリデーション → DB保存
    │
    ├─ [3] 工程配賦データ取り込み
    │       └── 配賦Excel（複数ファイル可） → アップロード → バリデーション → DB保存
    │
    ├─ [4] 労務費振替データ取り込み
    │       ├── 振替依頼書Excel（複数ファイル可） → アップロード → バリデーション → DB保存
    │       └── 労務費単価の設定
    │
    ├─ [5] 応援連絡票取り込み
    │       ├── 応援連絡票Excel（複数ファイル可） → アップロード → バリデーション → DB保存
    │       └── 労務費振替Excel生成（応援データから）
    │
    ├─ [6] SAP出力
    │       ├── 工程配賦TSVダウンロード
    │       └── 労務費振替TSVダウンロード
    │
    └─ [7] 月次処理完了
```

### 5.2 Excelファイル取り込みフロー（共通）

```
ユーザー
    │ ファイル選択・アップロード
    ▼
[バリデーション]
    ├── ファイル形式チェック（.xlsx/.xls）
    ├── シート名チェック
    ├── ヘッダー行チェック
    └── 行単位データバリデーション（Pydantic）
         ├── エラーあり → エラー詳細表示（行番号・フィールド名・エラー内容）
         └── エラーなし → DB保存 → 完了メッセージ表示
```

### 5.3 マスタインポートフロー（確認あり）

```
管理者
    │ マスタExcelアップロード
    ▼
[バリデーション・件数チェック]
    ├── エラーあり → エラー表示
    └── 正常
         ├── 件数増加/同数 → 即時保存
         └── 件数減少 → 警告表示 → 確認ボタン → 全件置換保存
```

### 5.4 SAP出力フロー

```
ユーザー
    │ SAP出力ボタン押下
    ▼
[データ取得・計算]
    ├── 工程配賦：DBデータ → TSV変換
    └── 労務費振替：DBデータ × 労務費単価 → 振替金額算出 → TSV変換
    │
    ▼
[ダウンロード]
    TSVファイル（CP932エンコード、ファイル名にYYYYMM付与）
```

## 7. 非機能要件

### 7.1 セキュリティ要件

| 要件ID | 要件 | 内容 |
|--------|------|------|
| SEC-01 | 認証 | 全ページに対してログイン必須。未認証時はログインページへリダイレクト |
| SEC-02 | パスワード保護 | パスワードはWerkzeug `generate_password_hash` でハッシュ化。平文保存禁止 |
| SEC-03 | CSRF対策 | Flask-WTFによるCSRFトークン検証を全フォームに適用 |
| SEC-04 | 一時ファイル管理 | アップロードされたExcelファイルは処理直後に削除し、サーバー上に残存させない |
| SEC-05 | セッション管理 | セッションベース認証（JWT不使用）。ログアウト時にセッションを破棄 |
| SEC-06 | 入力バリデーション | Pydanticによる型・必須チェックにより不正データのDB保存を防止 |
| SEC-07 | SQLインジェクション対策 | SQLAlchemy ORMを使用し、生SQLは使用しない |

### 7.2 ユーザビリティ要件

| 要件ID | 要件 | 内容 |
|--------|------|------|
| UX-01 | エラーメッセージの明確性 | バリデーションエラーは行番号・フィールド名・エラー内容を含めて表示 |
| UX-02 | レスポンシブ対応 | Bootstrap 5を使用し、PC・タブレット等の主要解像度に対応 |
| UX-03 | 部分更新 | HTMX を使用し、ページ全体リロードなしで一覧・モーダルを更新 |
| UX-04 | アコーディオン形式フォーム | アップロードフォームはアコーディオンで折りたたみ可能 |
| UX-05 | 複数ファイルアップロード | 配賦・労務費・応援モジュールはドラッグ&ドロップによる複数ファイル同時アップロードに対応 |
| UX-06 | モーダル表示 | データ詳細・削除確認はモーダルダイアログで表示 |

### 7.3 性能要件

| 要件ID | 要件 | 内容 |
|--------|------|------|
| PER-01 | ファイル処理時間 | 通常規模のExcelファイル（1,000行以内）のアップロード・バリデーション・保存を30秒以内に完了 |
| PER-02 | ページネーション | 大量データ時もページネーション（20件/ページ）によりページ表示速度を確保 |

### 7.4 可用性・保守性要件

| 要件ID | 要件 | 内容 |
|--------|------|------|
| AVL-01 | ログ出力 | アプリケーションログをファイルおよびコンソールに出力し、障害調査を可能にする |
| AVL-02 | エラーページ | 403/404/500エラーに対してカスタムエラーページを表示 |

## 8. 制約条件・前提条件

### 8.1 制約条件

| No | 制約 |
|----|------|
| 1 | Excelファイルは `.xlsx` または `.xls` 形式に限定 |
| 2 | SAP出力ファイルはTSV形式、CP932（Windows-31J）エンコードで固定 |
| 3 | ユーザー登録はスクリプト（`scripts/create_admin.py`）で行い、Web画面からの登録は行わない |
| 4 | JWTは使用せず、セッションベース認証とする |
| 5 | アップロードされたExcelファイルはサーバー上に保存しない |

### 8.2 前提条件

| No | 前提 |
|----|------|
| 1 | Excelファイルは所定のフォーマット（シート名・ヘッダー行）に従って作成されている |
| 2 | 処理は月次単位で行い、月の切り替え時に全データクリアを実施する |
| 3 | 労務費振替のSAP出力には事前に当月の労務費単価が設定されている必要がある |
| 4 | マスタデータ（課マスタ・変換マスタ・地区・勘定科目・原価センタ）は事前に登録済みである |
| 5 | ネットワーク環境はイントラネット内に閉じた運用を想定 |

## 9. 用語集

| 用語 | 説明 |
|------|------|
| バッチ | 1回のExcelファイルアップロードで取り込まれたデータのまとまり |
| 工程配賦 | 製造工程・プロジェクトへのコスト配賦データ |
| 労務費振替 | 労務費を適切な原価センタへ振り替えるための依頼データ |
| 応援連絡票 | 他地区への応援派遣を記録した文書 |
| 変換マスタ | 人事所属コードから原価センタ・勘定科目コードへの変換テーブル |
| 処理年月 | 現在の月次処理対象年月（YYYY-MM形式） |
| SAP | 本社基幹システム（ERP）。本システムはSAP入力用TSVファイルを生成する |
| CP932 | Windows日本語環境のShift_JIS拡張文字コード。SAP出力ファイルのエンコードに使用 |
| HTMX | サーバーサイドHTMLの部分更新を実現するJavaScriptライブラリ |
| Pydantic | Pythonのデータバリデーションライブラリ。Excel読み込みデータの型・必須チェックに使用 |
