# エクセルデータ取り込みシステム

# 概要
エクセルデータ取り込みシステムは、ユーザーがエクセルファイルをアップロードし、そのデータをシステム内で処理・保存するためのアプリケーションです。主な機能には、ファイルのアップロード、データのバリデーション、データベースへの保存、およびエラーハンドリング、データのテーブル表示が含まれます。
ただし、ファイルは保存されず、アップロードされたファイルは処理後に削除されます。
JWTは使用せず、セッションベースの認証を採用します。ユーザーは管理者によって事前にデータベースに登録されるため、登録機能は不要です。

# 機能要件
1. **ファイルアップロード**
   - ユーザーはエクセルファイルを選択してアップロードできる。
   - アップロードされたファイルはサーバーに一時的に保存されるが、処理後に削除される。
2. **データバリデーション**
   - アップロードされたエクセルファイルのデータは、Pydanticを使用してバリデーションされる。
   - バリデーションエラーが発生した場合、ユーザーにエラーメッセージが表示される。
3. **データベース保存**
   - バリデーションが成功したデータは、SQLite（開発環境）またはSQLServer（本番環境）に保存される。
4. **エラーハンドリング**
   - ファイルのアップロードやデータ処理中にエラーが発生した場合、適切なエラーメッセージがユーザーに表示される。
5. **データのテーブル表示**
   - 保存されたデータは、フロントエンドでテーブル形式で表示される。
   - テーブルはページネーションやソート機能を備える。
6. **ログイン**
   - ユーザーはログインしてシステムにアクセスする必要がある。
   - ログイン情報はセキュアに管理される。
   - 登録機能は不要で、ユーザーは管理者によって事前にデータベースに登録される。


# 非機能要件
1. **セキュリティ**
   - アップロードされたファイルは、サーバーに保存されないため、セキュリティリスクが低減される。
   - バリデーションにより、不正なデータの保存を防止する。
3. **ユーザビリティ**
   - ユーザーインターフェースは直感的で使いやすいことが求められる。
   - エラーメッセージはわかりやすく、読み込んだデータの何行目でエラーが発生したかを示すことで、ユーザーが問題を理解しやすいようにする。

# コンポーネント
flaskのマクロを使用して、ファイルアップロードフォームやデータテーブルのコンポーネントを作成することができます。これにより、コードの再利用性が向上し、保守性も高まります。
- ファイルアップロードフォームコンポーネント
	- ドラッグアンドドロップエリア
	- ドラッグアンドドロップエリアクリックでファイル選択ダイアログを開く機能

	- ファイル選択後、選択されたファイルのリストを表示する機能
	- ファイルリストからファイルを削除する機能
- データテーブルコンポーネント(htmxを使用して、ページネーションやソート機能を実装)
	- テーブルヘッダー
	- テーブルボディ
	- ページネーションコントロール
	- ソート機能
	- フィルタリング機能
	- ボタンを使用して、データの詳細を表示するモーダルを開く機能
	- ボタンを使用して、データの削除確認モーダルを開く機能
- エラーメッセージ表示コンポーネント
- モーダルコンポーネント（エラー詳細表示用）
- モーダルコンポーネント（データ詳細表示用）
- モーダルコンポーネント（コンファーム用）

# エクセルファイル
## フォーマット
エクセルファイルは、1行目にヘッダーがあり、2行目以降にデータが入力されている形式のファイルと、ヘッダのレンジが指定されている形式のファイルの両方に対応する必要があります。ヘッダのレンジが指定されている場合、システムはそのレンジを読み取ってヘッダを特定し、データの開始行を判断します。
ファイルのフォーマットが変わる可能性を考慮して、ファイルのフォーマット情報をyamlファイルとして管理する。
```yaml
salary:
	display_name: 給与データ
	sheet_name: 給与データ
	header_row: 1
	use_cols: A:G
	columns: 
		- name: 部署コード
		  type: str
		- name: 本給
		  type: int
		- name: 能力給
		  type: int
		- name: 報酬
		  type: int
		- name: 手当
		  type: int
		- name: 人員数
		  type: int
		- name: 給与額
		  type: int

allocation:
	display_name: 配賦データ
	sheet_name: 配賦データ
	header_row: 2
	use_cols: A:C
	columns: 
		- name: 地区コード
		  type: str
		- name: 課コード
		  type: str
		- name: 按分人員数
		  type: float

```

# データベース
## 必須テーブル
- ユーザーテーブル
	- id (int, primary key)
	- username (str, unique)
	- password (str)
	- created_at (datetime)
	- updated_at (datetime)

- アップロードされたファイルのテーブル(バッチとして括るためのテーブル)
	- id (int, primary key)
	- file_name (str)
	- file_type (str): 給与データ、配賦データ、労務費振替データなど
	- created_at (datetime)
	- created_by (int, foreign key to user table)

## 必須カラム

- created_by (int, foreign key to user table)
